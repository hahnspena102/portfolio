BreadQuest II: Into the Oven

BreadQuest II: Into the Oven is a web-based top-down dungeon crawler video game focused on thoughtful decision-making and skillful maneuvering.

Play Now!

Join Sir Gluten, a brave bread knight, on his perilous quest to save the villagers of Dough Village from their doom! In order to rescue his people, Sir Gluten traverses the depths of a dungeon where he encounters a cast of dangerous, medley of dessert/sweet treat-themed enemies. The deeper and deeper he journeys, the more he questions the dungeon. Why are there so many monsters? Why are they working together? What do they want with the villagers? Beat the game to uncover the mystery.

Technical Contributions

Enemies & Items

Designed using ScriptableObjects and a modular inheritance system. A parent Enemy class enabled child enemies to share core functionality while allowing easy tuning of stats like health, damage, and movement via data-driven design.

Inventory System

Built a persistent inventory system to maintain weapons, coins, and level progress across scenes. Each weapon had custom properties (damage, animation, fire style), which made system-wide persistence and flexibility challenging but rewarding.

Boss Battle (OVN-9000)

Implemented using a simple Finite State Machine to manage five distinct attack states and phase-based unlocks. This made the behavior logic structured but complex due to OVN’s variety of attacks.

Tools

Unity - chosen due to our prior experience with the engine, which helped streamline collaboration

Aseprite - creating and animating 2D pixel art assets such as characters, weapons, and UI elements.

Git & GitHub - version control and collaboration. Enabled smooth team workflow with branch management, pull requests, and issue tracking.

Visual Studio Code - primary code editor for C# scripting within Unity, supporting debugging and integration with Unity's tools.
